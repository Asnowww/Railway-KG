# KG知识图谱平台 项目启动与编译说明

## 1. 项目位置

- 项目根目录：`C:\Users\patri\OneDrive - bjtu.edu.cn\Files\Code\KG-Frontend\shadboard\starter-kit`
- 技术栈：`Next.js 15`、`React 19`、`TypeScript`、`pnpm`

## 2. 环境要求

- `Node.js >= 22`
- `pnpm >= 10`
- 建议在 `PowerShell` 中执行命令

可先检查版本：

```powershell
node -v
corepack pnpm -v
```

## 3. 首次安装依赖

如果是第一次拉取项目，需要先安装依赖：

```powershell
cd "C:\Users\patri\OneDrive - bjtu.edu.cn\Files\Code\KG-Frontend\shadboard\starter-kit"
corepack pnpm install
```

## 4. 启动开发环境

开发模式用于本地调试页面，支持热更新：

```powershell
cd "C:\Users\patri\OneDrive - bjtu.edu.cn\Files\Code\KG-Frontend\shadboard\starter-kit"
corepack pnpm dev
```

启动后默认访问：

- 首页：[http://localhost:3000](http://localhost:3000)
- 知识图谱工作台：[http://localhost:3000/workbench](http://localhost:3000/workbench)

在完成构建后，可以用生产模式启动：

然后在浏览器中打开：

- [http://localhost:3000](http://localhost:3000)

## 7. 常用辅助命令

### 7.1 代码格式化

```powershell
corepack pnpm format
```

### 7.2 查看 3000 端口是否被占用

```powershell
Get-NetTCPConnection -LocalPort 3000 -State Listen
```

### 7.3 停止占用 3000 端口的进程

先查出进程号：

```powershell
Get-NetTCPConnection -LocalPort 3000 -State Listen | Select-Object OwningProcess
```

再结束进程：

```powershell
Stop-Process -Id <进程号>
```

## 8. 环境变量说明

当前项目已配置的 `.env` 内容如下：

```env
BASE_URL=http://localhost:3000
HOME_PATHNAME=/dashboards/analytics
```

说明：

- `BASE_URL`：本地访问地址
- `HOME_PATHNAME`：模板预留字段，当前首页实际使用的是 `/`

## 9. 常见问题

### 9.1 执行 `pnpm` 失败

可先执行：

```powershell
corepack enable
```

然后重新运行相关命令。

### 9.2 3000 端口已被占用

先结束占用端口的旧进程，再重新执行：

```powershell
corepack pnpm dev
```

或：

```powershell
corepack pnpm start
```

### 9.3 构建通过但页面没有更新

可能是旧开发服务仍在运行。处理方式：

1. 停掉旧的 `node` 进程
2. 重新执行 `corepack pnpm dev`
3. 刷新浏览器页面

## 10. 推荐使用顺序

日常开发建议按以下顺序执行：

```powershell
corepack pnpm install
corepack pnpm dev
corepack pnpm build
```

如果只想本地查看最终构建效果，则执行：

```powershell
corepack pnpm build
corepack pnpm start
```
